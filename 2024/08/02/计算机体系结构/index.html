计算机系统概述

操作系统的基本特征

操作系统的基本特征: 并发，共享，虚拟，异步

操作系统最基本的两个特征：并发，共享

对处理机管理就是对进程的管理

接口

命令接口

联机命令接口

交互式命令接口,适用于分时或者实时 操作系统

脱机命令接口

又叫批处理接口,适用于批处理系统,由一组作业控制命令构成.脱机用户不能直接干预

我要是老师就估计在这弄混淆,说批处理接口适用于分时

程序接口

程序接口由一组系统调用或者叫广义指令.用户在程序中使用系统调用获得各种服务

使用外部设备,申请分配回收内存 都用到系统调用

tips

没有任何软件支持的计算机叫裸机,在最内层,外面一层才是操作系统,覆盖了软件的叫扩充机器或者说虚拟机

操作系统发展历程

手工作业阶段(此时还没操作系统)

特征:所有工作都需要人工干预

缺点:用户独占全机资源利用率低,CPU等手工操作

批处理阶段(操作系统出现了)

单道批处理

内存始终保持一道作业，解决人机矛盾以及CPU和io设备速度不匹配的矛盾

自动性：无需人干预

顺序性: 按顺序

单道性：一次只有一个

多道批处理

宏观上并行,微观上串行,计算机内存中同时存放多道相互独立的程序

我要是老头就考说程序都放在磁盘里

优点: 资源利用率高,系统吞吐量大

缺点:用户相应时间长,不具备人机交互能力

就是用户不大能控制

分时操作系统

分时系统:把处理器的运行时间分成很多小片,轮流给人用.

分时os:多个用户通过终端同时共享一台主机

特征

  1. 同时性/多路性: 即大家都能一起用

  2. 交互性: 用户能控制计算机,多道批处理就控制不了

  3. 独立性:大家互不干扰

  4. 及时性:用户的请求可以再很短时间内获得相应

    如果扣字眼的话.第四个可能是考点

实时操作系统

硬实时

时间绝对保证,如飞机自动控制系统

软实时

偶尔可以违反时间规定,订票系统,银行管理系统

操作系统运行环境

处理器运行模式

特权指令

IO指令,置中断指令,取存用于内存保护的寄存器,送程序状态字到程序状态字寄存器

非特权指令

用户可以直接使用,不能直接访问系统的软硬件资源.仅仅可以访问用户的地址空间

依赖中断的服务

键盘输入,进程管理和调度,系统功能的调用,设备驱动,文件访问

中断服务只有一小部分属于内核

中断

这里以按下键盘为例

设备控制器

我们的电脑设备可以接非常多的输入输出设备，比如键盘、鼠标、显示器、网卡、硬盘、打印机、音响等等，每个设备的用法和功能都不同，那操作系统是如何把这些输入输出设备统一管理的呢?

为了屏蔽设备之间的差异，每个设备都有一个叫设备控制器（Device Control） 的组件，比如硬盘有硬盘控制器、显示器有视频控制器等。

设备控制器里有芯片，它可执行自己的逻辑，也有自己的寄存器，用来与 CPU 进行通信，比如：

通过写入这些寄存器，操作系统可以命令设备发送数据、接收数据、开启或关闭，或者执行某些其他操作。
通过读取这些寄存器操作系统可以了解设备的状态，是否准备好接收一个新的命令等

控制器是有三类寄存器，它们分别是状态寄存器（Status Register）、 命令寄存器（Command Register）以及数据寄存器（Data Register）

  1. 数据寄存器，CPU 向 I/O 设备写入需要传输的数据，比如要打印的内容是「Hello」，CPU 就要先发送一个 H 字符给到对应的 I/O 设备。
  2. 命令寄存器，CPU 发送一个命令，告诉 I/O 设备，要进行输入/输出操作，于是就会交给 I/O 设备去工作，任务完成后，会把状态寄存器里面的状态标记为完成。
  3. 状态寄存器，目的是告诉 CPU ，现在已经在工作或工作已经完成，如果已经在工作状态，CPU 再发送数据或者命令过来，都是没有用的，直到前面的工作已经完成，状态寄存标记成已完成，CPU 才能发送下一个字符和命令。


当用户输入了键盘字符，键盘控制器就会产生扫描码数据，并将其缓冲在键盘控制器的寄存器中，紧接着键盘控制器通过总线给 CPU 发送中断请求。

CPU 收到中断请求后，操作系统会保存被中断进程的 CPU 上下文，然后调用键盘的中断处理程序。

键盘的中断处理程序是在键盘驱动程序初始化时注册的，那键盘中断处理函数的功能就是从键盘控制器的寄存器的缓冲区读取扫描码，再根据扫描码找到用户在键盘输入的字符，如果输入的字符是显示字符，那就会把扫描码翻译成对应显示字符的 ASCII 码，比如用户在键盘输入的是字母 A，是显示字符，于是就会把扫描码翻译成 A 字符的 ASCII 码。

得到了显示字符的 ASCII 码后，就会把 ASCII 码放到「读缓冲区队列」，接下来就是要把显示字符显示屏幕了，显示设备的驱动程序会定时从「读缓冲区队列」读取数据放到「写缓冲区队列」，最后把「写缓冲区队列」的数据一个一个写入到显示设备的控制器的寄存器中的数据缓冲区，最后将这些数据显示在屏幕里。

显示出结果后，恢复被中断进程的上下文。
